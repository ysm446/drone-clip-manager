### 2026-07-09 00:47
- **しばらく使う（特にシークを繰り返す）と動画が応答しなくなる問題を修正**。
  - 原因: mpv の stdout/stderr をパイプに繋いだまま誰も読んでいなかったため、ログが OS の
    パイプバッファ（約64KB）を埋めた時点で mpv の書き込みがブロックし、プレイヤー全体が停止していた。
  - 修正: `--no-terminal` でターミナル出力を停止し、stdio を切り離した。
- **mpv が落ちても自動復帰するようにした**。mpv プロセス / IPC の異常を検知したら mpv を再起動し、
  同じ動画を同じ再生位置から自動で開き直す（短時間に連続で落ちる場合は `<video>` 再生へフォールバック）。
  従来は一度異常が起きるとアプリを再起動するまで映像を制御できなくなっていた。
