# ドローン映像 構成・編集のコツ

作成日時: 2026-07-17 15:46
更新日時: 2026-07-17 15:51

ドローン空撮映像を「どう並べるか」「1 カットを何秒にするか」についての知見メモ。
本アプリ（クリップのブックマーク・書き出し）で素材を組み立てるときの指針として参照する。

> 位置づけ: 実装仕様ではなく、映像編集の考え方の資料。
> クリップの in/out やシーケンス編集を行うときの判断材料として使う。

---

## 前提: 「いい景色」より「並べ方」

ドローン映像は、1 カット単体の美しさよりも **カット同士の流れ** で印象が大きく変わる。
特に YouTube では、単体の絶景よりも「つながり」が最後まで見てもらえるかを左右する。

---

## 1. 並べ方の 7 つの原則

### ① 大 → 小 → 大 を意識する（スケールの変化）

ずっと上空からの映像だと単調になる。視点のスケールを変えると飽きさせない。

- 超広角の全景
- 少し低くして建物や道路を見せる
- 被写体に寄る
- また引いて景色を見せる

### ② 動きの方向を揃える

連続するカットの飛行方向をそろえると気持ちよく見える。

- 左 → 右へ飛ぶ
- 次も左 → 右
- 次も左 → 右

逆方向にすると一瞬「どこへ行った？」という印象になる。
ただし **場面転換では逆方向をあえて使う** と効果的。

### ③ 高度を徐々に変える

高度を段階的に下げると、視聴者がその場所へ近づいている感覚になる。

```
100m
 ↓
70m
 ↓
40m
 ↓
15m
```

逆に **最後に一気に上昇** すると締め（エンディング）になる。

### ④ 動きの種類を続けない

同じ動きが続くと単調になる。

- ✗ 前進 → 前進 → 前進 → 前進
- ◎ 前進 → パン → 上昇 → 横移動 → Orbit

動きの種類を変えると映像にリズムが生まれる。

### ⑤ 速度にもリズムを付ける

全部ゆっくりだと眠くなる。速度に「波」を作る。

- ゆっくり
- 少し速い
- ゆっくり
- ダイナミック

### ⑥ 「旅」を作る（最重要）

きれいな映像を並べるだけより、**実際に移動している感覚** を作ると、
視聴者は映像を「体験」として楽しめる。

```
駐車場 → 道路 → 峠 → 展望台 → 山頂
```

### ⑦ 最初の 15 秒が勝負

YouTube では冒頭に

- 一番の絶景
- 一番の迫力
- 一番印象的なカット

を持ってくることが多い。**時系列どおりである必要はない。**

## 2. 構成例（山岳ドローン映像）

```
最高の絶景（5 秒）
 ↓
タイトル
 ↓
遠景で場所紹介
 ↓
徐々に近づく
 ↓
谷や道路をたどる
 ↓
湖や橋などのアクセント
 ↓
山頂付近
 ↓
一番迫力のある低空飛行
 ↓
上昇してエンディング
```

## 3. 視線誘導（スイス・アルプス題材で特に有効）

画面内に自然な「線」があるカットをつなぐと、視聴者の目が迷わず、
次のカットへも自然につながる。

- 道路
- 鉄道
- 川
- 尾根

アルブラ線やフルカ峠のようなロケーションは、この視線誘導を活かしやすく、
ドローン映像との相性がとても良い。

---

## 4. 1 カットの秒数

ドローン映像は地上カメラより **少し長めが基本**。カメラも景色も動くため長尺でも成立する。

### 目安

| 1 カットの長さ | 印象 | おすすめ度 |
|---|---|---|
| 2〜3 秒 | 忙しい・PV 風 | △ |
| 4〜6 秒 | テンポが良い | ◎ |
| 7〜10 秒 | 景色をじっくり見せる | ◎ |
| 10〜15 秒 | 雄大さ・没入感 | ○ |
| 15 秒以上 | よほど変化がないと長く感じる | △ |

**一番のおすすめは 5〜8 秒**（表の ◎ 2 区分にまたがる中心帯）。
景色を認識する時間があり、次のカットも待ち遠しく感じられる。

### 動きによる違い

- **速いカット**（低空飛行・道路沿い）: 4〜6 秒でも十分
- **ゆっくりしたカット**（雲海・朝焼けなど動きが少ない）: 8〜12 秒でも飽きない
- **ホバリングだけ**の映像は、5 秒でも長く感じる

### 編集テクニック: 前後を切る

20 秒飛ばした素材でも、

- 最初の 2 秒（加速・姿勢が安定していない部分）
- 最後の 2 秒（停止・旋回に入る部分）

をカットして、見どころの **6〜8 秒だけ** 使うことが多い。

> 本アプリの「in/out を 1 キーフレームずつずらすボタン」は、
> この前後の微調整に使える。

### YouTube でよく見る構成（10 分前後の場合）

- 5〜6 秒のカットを約 50 本
- 8〜10 秒のカットを約 30 本
- 途中に 15 秒程度の「見せ場」を数本

合計するとおよそ 9〜10 分になる（5 分の映像なら本数をおおよそ半分にする）。
このリズムが、最後まで飽きずに見てもらいやすい。

---

## 5. まとめのコツ: 余韻を残す

「景色がわかった瞬間に切り替える」のではなく、
**「もう少し見たい」と感じる少し手前で切り替える** のがコツ。

視聴者に少し余韻を残すテンポの方が、次のカットへの期待感が自然につながる。

## 6. アプリでの活用（機能案）

本資料の原則は「クリップのメタデータ + シーケンスの並び」に対するルールとして
機械的に表現できるものが多く、既存機能（自由タグ・ノードグラフのシーケンス・
ノードの秒数表示）と素直につながる。実装状況は
[../plan/progress.md](../plan/progress.md) を参照。

### 6.1 構成タグの語彙（既存タグ機構の延長）

名前空間付きタグを推奨語彙として補完に出す。既存の自由タグ（`segment_tags`）の
ままで始められ、Phase 2.8 の残タスク「カテゴリ（名前空間）」と重なる。

| 名前空間 | 値の例 | 対応する原則 |
|---|---|---|
| `motion:` | 前進 / パン / 上昇 / 横移動 / orbit / ホバリング | ④ 動きの種類 |
| `dir:` | L→R / R→L / 前 / 後 | ② 方向を揃える |
| `scale:` | 遠景 / 中景 / 寄り | ① 大→小→大 |
| `speed:` | ゆっくり / 普通 / 速い | ⑤ 速度のリズム |
| `alt:` | 高 / 中 / 低（または概算 m） | ③ 高度変化 |

### 6.2 シーケンスの「構成リント」

ノードグラフの順路に対して原則をチェックし、警告や補助線を表示する。

- ④: 同じ `motion:` タグが 3 連続 → 「動きが単調」警告
- ②: `dir:` が交互に反転 → 「方向が揃っていない」警告
- 秒数が 4 秒未満 / 15 秒超のノード → 注意マーク
- ⑦: 冒頭ノードの合計が 15 秒を超える位置に「ここまでが勝負」ライン

### 6.3 秒数バッジの色分け（最小の一手）

ノードにはすでに秒数を表示済みなので、着色だけで成立する。

- 4〜10 秒 = 通常
- 10〜15 秒 = 淡色
- 4 秒未満 / 15 秒超 = 警告色

### 6.4 「前後 2 秒トリム」プリセット

既存の in/out キーフレームナッジの隣に、「頭とお尻を約 2 秒詰める」
ワンクリック操作を置く（§4 の編集テクニックそのまま。実際はキーフレーム
スナップの都合で「約 2 秒」になる）。

### 6.5 Phase 3（カメラモーション自動解析）との関係

spec が拡張点として残しているカメラモーション自動解析は、6.1 のタグを
自動生成する話に相当する。解析で `motion:` / `dir:` / `speed:` が自動付与
できれば、6.2 のリントが手作業ゼロで機能する。
**本資料は「Phase 3 で何を検出すべきか」の要件定義としても読める。**
