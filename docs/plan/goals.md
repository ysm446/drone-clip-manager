# goals — flight-cut の目的と完成形

作成日時: 2026-07-08 12:29
更新日時: 2026-07-08 12:29

このドキュメントは、プロジェクトの目的・完成形・重視する価値をまとめる。
詳細な機能仕様は [../spec.md](../spec.md) を正本とする。

## 目的

ドローンで撮った大量の 4K 素材を、**素材を劣化させずに** 整理・区間マーク・切り出しするための、
ローカル完結のデスクトップアプリを作る。

## 中核となる価値（3つ）

1. **管理** — ドローン素材（DJI 等の分割ファイル、HEVC/H.264 10bit、高fps）を1画面で俯瞰する。
2. **区間ブックマーク** — 1本の動画に複数の in/out 区間をラベル付きで記録・保存する。非破壊。
3. **ロスレス書き出し** — 記録した区間を `-c copy`（stream copy）で個別ファイルに切り出す。再圧縮しない。

## 完成形（MVP のゴール）

- **Phase 2 完了時点** を MVP とする。すなわち以下が一通り動く状態:
  - ルートフォルダを指定してライブラリを取り込み、一覧・検索・フィルタできる。
  - 動画の区間をキーフレームスナップ付きでブックマークできる。
  - ブックマークした区間をロスレス（stream copy）で個別 mp4 に書き出せる（単発 / バッチ）。
- ライブラリのメタデータは **ルートフォルダ直下の `.flightcut/` に自己完結** して保存され、フォルダごと可搬。

## 重視する価値・判断の軸

- **非破壊** — 元素材には一切手を触れない。区間はメタデータとしてのみ持つ。
- **ロスレス最優先** — 書き出しは再エンコードしない（stream copy）。フレーム精度より無劣化を優先する。
  - その代償として **カット点はキーフレーム（GOP 境界）単位** になる。これは仕様として UI で明示する。
- **ローカル完結・可搬** — クラウド同期なし。ルートフォルダを移動しても壊れない相対パス管理。
- **最小構成** — LLM も Python 解析も今回は無いので、バックエンドサイドカーを持たず Electron メインプロセスから直接 ffmpeg を叩く。
- **拡張の下地だけ残す** — カメラモーション自動解析（前進/後進/横スライド/回転タグ）を後付けできるよう、
  データモデルと処理層に拡張点のみ用意する（実装はしない）。

## 今回のスコープ外（拡張点だけ残す）

- カメラモーション自動解析（オプティカルフロー / VLM）
- 分割ファイルを跨いだ連結切り出し（グルーピング表示までは検討可、跨ぎ書き出しは Phase 3）
- クラウド同期 / 共有
- カラーグレーディング / 変換書き出し

## 対象素材の特性（ドローン特化）

- **分割ファイル**: `DJI_0001.MP4` … のように数分単位で自動分割される。`flight_group` で1フライトに束ねたい。
- **コーデック**: HEVC 10bit、D-Log / HLG など。copy なのでプロファイルはそのまま維持。
- **高fps**: 4K60 / 4K120。fps はメタから正確に取り、区間長計算・表示に反映。
- **メタデータ**: 可能なら GPS・撮影日時を抽出し、ソート / フィルタに使う。
